# Package Lists

This directory contains lists of installed packages for easy restoration on a new system.

## Files

- `apt.txt` - APT/Debian packages
- `apt-selections.txt` - Detailed APT package selections
- `dnf.txt` - DNF/Fedora packages  
- `pacman.txt` - Pacman/Arch packages
- `aur.txt` - AUR packages (Arch, requires yay)
- `flatpak.txt` - Flatpak applications
- `snap.txt` - Snap packages
- `pip.txt` - Python pip packages
- `npm.txt` - NPM global packages
- `cargo.txt` - Rust cargo binaries

## Usage

### Automatic Backup

These files are automatically generated by the backup script:
```bash
./scripts/backup.sh --packages
# or
make backup-packages
```

### Automatic Installation

Install all packages on a new system:
```bash
./scripts/install-packages.sh
# or
make install-packages
```

### Manual Creation

If you don't have packages backed up yet, you can create them manually using the example files:

```bash
cp pacman.txt.example pacman.txt
cp aur.txt.example aur.txt
# Edit the files to add your packages
```

## Package File Format

- One package name per line
- Lines starting with `#` are comments (ignored)
- Empty lines are ignored
- For pip: you can specify versions like `package==1.2.3`

## Manual Installation

### Pacman (Arch Linux)
```bash
sudo pacman -S --needed - < pacman.txt
```

### AUR (requires yay)
```bash
yay -S --needed - < aur.txt
```

### APT (Debian/Ubuntu)
```bash
xargs sudo apt install -y < apt.txt
```

### Flatpak
```bash
cat flatpak.txt | xargs -n1 flatpak install -y flathub
```

### pip
```bash
pip3 install --user -r pip.txt
```

## Important Notes

### AUR Packages
AUR packages require `yay` or another AUR helper. The install script will offer to install yay automatically.

### PyCharm
Use the dedicated PyCharm installation script:
```bash
./scripts/install-pycharm.sh
# or
make install-pycharm
```

### GNOME Extensions
GNOME extensions are managed separately. See `gnome/extensions/` directory.

## Troubleshooting

If a package fails to install:
1. Check if the package exists in the repository
2. Remove the problematic package from the list
3. Install it manually later
4. Check the installation logs for errors
